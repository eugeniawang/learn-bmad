# COMPETENCY_MAP.md — Learn BMAD

> Maps the course's four goals to the modules that build them, the scenario checks that
> prove them, and the `SOURCE.md` anchors they trace to. The instructor uses this to log
> `competency_progress` in `progress.json`. Because the course is non-sequential, a
> competency is "met" once *any* module that builds it is completed with a passing check.

| ID | Competency (goal) | Built in | Scenario check | Source |
| --- | --- | --- | --- | --- |
| **C1** | Explain *why* BMAD beats one-chat vibe coding | M1 | "Here's a vague one-chat build that went sideways — name which of the five beliefs would have caught it." | SOURCE §1 |
| **C2** | Name the four phases and what each is *for* | M2, M4, M5, M6 | "Given a half-finished project, which phase produced this artifact / which is missing?" | SOURCE §2 |
| **C3** | Tell Planning from Solutioning (what/why vs how/units) | M4, M5 | "Sort these statements into What-&-Why vs How-&-units-of-work." | SOURCE §2.2–2.3 |
| **C4** | Pick the right agent for a job; know when to convene Party Mode | M3 | "Who do you call first for *this* problem — and is it a one-agent or a Party-Mode question?" | SOURCE §3 |
| **C5** | Right-size the process (quick-dev vs full method) | M7 | "Three changes of different sizes — quick-dev or full method for each, and why?" | SOURCE §4 |
| **C6** | Know where to get unstuck (BMad-Help + docs) without guessing | M8 | "You're lost mid-project — what's your first move, and where would you look it up?" | SOURCE §5, §9 |
| **C7** | Map a real build onto BMAD end-to-end | M9 (capstone), draws on all | The capstone one-page plan itself is the proof. | SOURCE §2, §4 |
| **C8** | Understand how BMAD extends itself — building agents (incl. memory) and skills | M10, M11 | "Skill, whole agent, or neither?" + "facts vs. memory?" judgment calls. | SOURCE §6, §6.5 |

## Remediation paths

- **C1 weak** → explain_again with one concrete "vibe coding gone wrong" story, then retry.
- **C2/C3 muddled** → guided_retry sorting real artifacts into phases; revisit the §2 table.
- **C4 weak** → re-run Party Mode role-play on the learner's own build.
- **C5 weak** → the "three changes" drill again with the learner's own backlog.
- **C6 weak** → walk the Docs Map (SOURCE §9) live, once.
