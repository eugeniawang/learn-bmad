# LESSONS.md — Index (the buffet menu)

> **This is an index, not the lessons.** Each module lives in its own file under
> `lessons/`. The instructor loads **only** the current module file — never this whole
> corpus. This course is **non-sequential**: pick any module, in any order.

## How every module is built (the shape)

Each `lessons/NN-*.md` file follows the same beats so a cold start always works:

1. **Where this fits** — a 2-line primer so you can start here with nothing else done.
2. **The point** — one sentence: what you'll walk away understanding.
3. **Do This First** — you do/decide something *before* any explanation.
4. **What Just Happened** — 2–3 lines making sense of it.
5. **The Idea** — the concept, short, in plain English.
6. **Pattern / Anti-pattern** — the good move, the common faceplant, how to tell them apart.
7. **Explain It Back** — you say it simply; the instructor patches gaps.
8. **Scenario Check** — one light "predict, then reveal" judgment call.
9. **Apply to Your Build** — update one line in `MY_BUILD.md`.
10. **Review Hooks / Go Further** — what to revisit; which module pairs well next.

## Cross-cutting guardrails (enforced every module)

- Fun, wry, short. No code shown. No setup rabbit-holes (point to the docs).
- Everything grounds in the learner's own build (`MY_BUILD.md`).
- Checks are light and scenario-based — never recall quizzes.
- Questions beyond the module → answer from `SOURCE.md` or point to the exact Docs Map
  entry (`SOURCE.md` §9). Never invent BMAD behavior.

## The principle spine (the five beliefs every module ties back to)

1. Plan before you build. 2. Make decisions explicit. 3. Scale rigor to the change.
4. Use specialists, not one generalist. 5. Keep the human in the loop where it matters.
